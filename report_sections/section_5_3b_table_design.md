## 5.4) Table Design

This section presents the exact structural design of every MongoDB collection used by Privacy Chat, derived directly from the Mongoose schema definitions in the source code. Each Mermaid `erDiagram` block reflects the precise field names, data types, key constraints, default values, and indexes as defined in the model files. The in-memory private message store is also documented for completeness.

---

### 5.4.1) users Collection
**Model file:** `backend/models/User.js`

```mermaid
erDiagram
    users {
        ObjectId _id        PK  "Auto-generated by MongoDB"
        String   username       "required, unique, trim, minlength:3, maxlength:20"
        String   email          "required, unique, trim, lowercase"
        String   password       "required, minlength:6, bcrypt-hashed"
        String   avatar         "default: empty string, URL to uploaded image"
        Boolean  isOnline       "default: false"
        ObjectId contacts       "Array of ObjectId, ref: User (mutual contacts)"
        String   publicKey      "default: empty string, ECDH JWK public key for E2E"
        Date     createdAt      "auto, timestamps:true"
        Date     updatedAt      "auto, timestamps:true"
    }
```

**Indexes:**
- `{ username: 1, unique: true }`
- `{ email: 1, unique: true }`

**Post-delete hooks:** `findOneAndDelete` and `deleteOne` cascade to remove this user from all other users' `contacts` arrays.

### 5.4.2) messages Collection
**Model file:** `backend/models/Message.js`

```mermaid
erDiagram
    messages {
        ObjectId _id           PK  "Auto-generated by MongoDB"
        String   conversationId     "index:true, for DMs: smallerUserId_largerUserId"
        ObjectId groupId            "ref: Group, null for DMs"
        ObjectId sender             "ref: User, required"
        String   text               "default: empty string, max 500 chars"
        String   fileUrl            "optional, URL to uploaded file"
        String   fileName           "optional, sanitised original filename"
        String   fileType           "optional, MIME type"
        Number   fileSize           "optional, bytes"
        ObjectId deletedFor         "Array of ObjectId, ref: User (soft delete)"
        String   encryptedKeys      "default: empty string, JSON map of memberId to wrapped AES key"
        String   status             "enum: sent|delivered|read, default: sent"
        ObjectId readBy             "Array of ObjectId, ref: User"
        Date     createdAt          "auto, timestamps:true"
        Date     updatedAt          "auto, timestamps:true"
    }
```

**Indexes:**
- `{ conversationId: 1 }` (inline `index: true` on field)

**Note:** Private messages are **never** stored here. They live exclusively in the RAM `Map` in `socket/privateStore.js`.

### 5.4.7) verificationcodes Collection
**Model file:** `backend/models/VerificationCode.js`

```mermaid
erDiagram
    verificationcodes {
        ObjectId _id        PK  "Auto-generated by MongoDB"
        String   email          "required, lowercase, trim"
        String   code           "required, 6-digit numeric OTP"
        Date     expiresAt      "required, default: Date.now + 10 minutes"
        Date     createdAt      "auto, timestamps:true"
        Date     updatedAt      "auto, timestamps:true"
    }
```

**Indexes:**
- `{ expiresAt: 1 }, { expireAfterSeconds: 0 }` — TTL index: MongoDB auto-deletes document when `expiresAt` is reached
- `{ email: 1 }` — for fast OTP lookup by email

### 5.4.8) Private Message Store (RAM — Not a MongoDB Collection)
**Source file:** `backend/socket/privateStore.js`

This is a JavaScript `Map<sessionId, messageObject[]>` held in volatile server RAM. It is **never** written to MongoDB or disk.

```mermaid
erDiagram
    privatestore_ram {
        String  sessionId  PK  "Map key — the active private session ID"
        String  _messageId     "crypto.randomUUID() — unique per message"
        Object  sender         "Object with _id and username of the sender"
        String  text           "Message text content"
        String  fileUrl        "optional — temporary path to ephemeral uploaded file"
        String  fileName       "optional — sanitised original filename"
        String  fileType       "optional — MIME type"
        Number  fileSize       "optional — size in bytes"
        Boolean isPrivate      "always true — triggers ephemeral UI mode on client"
        String  createdAt      "ISO 8601 timestamp string"
    }
```

**Destruction paths:** `Map.delete(sessionId)` is called on session end, socket disconnect, Beacon API page-unload, or server restart. Any associated files are deleted from disk via `fs.unlink()` concurrently.

```mermaid
    users ||--o{ sessions       : "starts (startedBy)"
    users }o--o{ groupsessions  : "participates in (participants[])"
    users ||--o{ groupsessions  : "starts (startedBy)"
    users }o--o{ users          : "connected via (contacts[])"
    groups ||--o{ messages      : "has group messages (groupId)"
    groups ||--o{ groupsessions : "has private sessions (groupId)"
```
